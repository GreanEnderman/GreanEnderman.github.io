# Getting Started with Next.js 14

Next.js 14 brings powerful new features to the table, especially with the App Router. Let's explore how you can leverage these capabilities.

## Why Next.js 14?

The latest version includes:

- **Improved Performance**: Faster builds and optimized routing
- **Server Components**: Better performance by default
- **Streaming**: Progressive page rendering
- **Built-in Optimization**: Image, font, and script optimization

## Creating Your First App

```bash
npx create-next-app@latest my-app --typescript --tailwind --app
```

This command sets up a new project with:
- TypeScript for type safety
- Tailwind CSS for styling
- App Router for routing

## Server vs Client Components

One of the key concepts in Next.js 14 is the distinction between server and client components.

### Server Components (Default)

```tsx
export default async function ServerComponent() {
  const data = await fetch('https://api.example.com/data');
  return <div>{data.title}</div>;
}
```

### Client Components

```tsx
"use client";

import { useState } from 'react';

export function ClientComponent() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

## Conclusion

Next.js 14 provides a powerful foundation for building modern web applications. Start exploring today!

# Understanding TypeScript Generics

Generics are one of TypeScript's most powerful features. They allow you to create reusable components that work with a variety of types while maintaining type safety.

## What are Generics?

Generics allow you to define types that are parameterized. This means you can write code that can work with different types without losing type information.

## Basic Syntax

Here's a simple example:

```typescript
function identity<T>(arg: T): T {
  return arg;
}

const num = identity<number>(42); // Type: number
const str = identity("hello"); // Type: string (type inference)
```

## Generic Interfaces

You can also use generics with interfaces:

```typescript
interface Box<T> {
  value: T;
}

const numberBox: Box<number> = { value: 42 };
const stringBox: Box<string> = { value: "hello" };
```

## Generic Constraints

Sometimes you want to constrain the generic type:

```typescript
interface Lengthwise {
  length: number;
}

function loggingIdentity<T extends Lengthwise>(arg: T): T {
  console.log(arg.length);
  return arg;
}
```

## Real-World Example

Here's a practical example with API responses:

```typescript
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

interface User {
  id: number;
  name: string;
  email: string;
}

async function fetchUser(): Promise<ApiResponse<User>> {
  const response = await fetch('/api/user');
  return response.json();
}
```

## Conclusion

Generics are essential for writing flexible, reusable TypeScript code. Master them to level up your TypeScript skills!
